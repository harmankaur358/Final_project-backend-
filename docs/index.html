Users

User management and authorization

Get user details by UID

Accessible to admin, manager, or the same user.

Authorizations:
bearerAuth
path Parameters
uid
required
string

User ID whose details you want to fetch

Responses

Local server

http://localhost:3000/api/v1/users/{uid}

Response samples

  • 200
Content type
application/json
{
  • "uid": "abc123",
  • "email": "user@example.com",
  • "role": "user"
}

Alerts

Create a new weather alert

Authorizations:
bearerAuth
Request Body schema: application/json
required
id
string

Unique identifier for the alert

locationId
required
string

ID of the location associated with the alert

type
required
string
Enum: "Storm" "Rain" "Snow" "Heat" "Cold"

Type of weather alert

description
required
string [ 5 .. 500 ] characters

Detailed description of the weather alert

severity
required
string
Enum: "Low" "Medium" "High" "Critical"

Level of alert severity

startTime
required
string <date-time>

ISO 8601 start time of the alert

endTime
required
string <date-time>

ISO 8601 end time of the alert

Responses

Local server

http://localhost:3000/api/v1/alerts

Request samples

  • Payload
Content type
application/json
{
  • "id": "alert_001",
  • "locationId": "loc_123",
  • "type": "Storm",
  • "description": "Severe thunderstorm expected with high winds and heavy rainfall.",
  • "severity": "High",
  • "startTime": "2025-11-09T10:00:00Z",
  • "endTime": "2025-11-09T18:00:00Z"
}

Create a new weather forecast

Authorizations:
bearerAuth
Request Body schema: application/json
required
id
string

Unique identifier for the forecast

locationId
required
string

ID of the location this forecast belongs to

temperature
required
number

Predicted temperature in Celsius

humidity
required
number [ 0 .. 100 ]

Humidity percentage (0–100)

windSpeed
required
number >= 0

Wind speed in km/h

date
string <date-time>

ISO 8601 date and time of the forecast
